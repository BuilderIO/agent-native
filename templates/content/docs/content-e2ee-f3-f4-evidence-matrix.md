## How to read this artifact

This is a repository-backed prerequisite, not a claim about the deployed system. “Plaintext” means server- or browser-readable application data, including JSON and base64 encodings that are not encryption. “Server-encrypted” means ciphertext protected by a deployment-held key and therefore still readable by the service. “Unknown” is deliberate where the repository does not prove production configuration, retention, backups, provider policy, or who currently has privileged access.

Protected classifications are **P** (primary vault content), **D** (content-derived or duplicated material), **M** (metadata that may still be sensitive), and **O** (operational material that can contain protected content depending on inputs). Reader sets below are code-path readers, not a production grant inventory.

Those require the deployment evidence called out below.

## Verification record

This evidence pass intentionally did not install dependencies or touch production credentials.

- `node_modules` is absent at repository root.
- `pnpm action db-check-scoping --format json` is therefore unavailable; the baseline audit records the exact failure as `sh: agent-native: command not found`.
- Pure Node repository guards that do not require installed packages remain eligible and are run with this artifact's commit verification.
- The [production preflight](./content-production-exposure-inventory-2026-07-16.md) verified the public Netlify site/deploy, healthy database connection, public shell cache, content-free missing-public-route behavior and anonymous rejection from an authenticated diagnostic. It read no production documents or share data.
- Production database identity, blob provider, OAuth state, A2A peers, extensions, Notion connections, visibility/share counts, logs, backups, model-provider settings and anonymous media behavior were not credentialed or inspected. Their F4 rows remain pending by construction.
