# GTM Context

## Company

**Builder.io** — the AI product development platform where your team and AI agents build, review, and ship with confidence. What Figma did for product design, Builder is doing for product development.

AI product development is a new way to build software. AI agents and every role on the product team work together in shared workflows to build, review, and ship production-ready code. Teams ship faster, at higher quality, without scaling engineering headcount.

Three pillars make this possible:
- **Context** — AI that builds on your real codebase, tech stack, and design system
- **Collaboration** — Every role on the product team builds and ships alongside AI agents
- **Trust** — Review, approval, and guardrails built into every change

## Products

### Fusion (AI Product Development Platform)
Design-to-code, AI-powered prototyping, and visual IDE. Designers and PMs turn prompts or Figma designs into production-ready UI built with their team's real design system and code components. Developers review and iterate in their workflow. Like Claude Code in the cloud — but collaborative and visual.

**Use cases:** Ideation & prototyping, design-to-code, design system management.
**Personas:** Design, Product, Engineering.

### Publish (Enterprise Headless CMS)
Visual headless CMS where marketing builds, publishes, and iterates on pages using code components without engineering. A/B testing, localization, personalization built in.

**Use cases:** Visual CMS, page builder, experimentation, localization.
**Personas:** Marketing, Content, Engineering.

## Value Proposition

**Elevator:** Builder uniquely combines an AI-powered design-to-code tool, a visual editor, and an enterprise CMS into a complete, high-velocity workflow. Teams go from idea to production 10x faster.

**Pain points we solve:**
- One dev, one agent, one thing at a time → Builder runs massively in parallel on cloud environments
- Front-end burden on engineers (pixel pushing, design redlines, Figma-to-code) → designers and PMs contribute directly
- Non-engineers locked out of contributing to code → visual editing makes code accessible to everyone
- QA and PR review drowning → built-in review, approval, and guardrails
- Engineers as the bottleneck → every role ships alongside AI agents

**Proof points:**
- Leading HR software company: cut beta delivery by 75% (two months → two weeks)
- Fortune 500 fintech (Intuit): 76% saved time, 84% satisfied with on-brand output
- Eagle Eye Solutions: 50% reduction in UI dev time, ~£115K annual savings
- CBRE: cut feature delivery by a full sprint; 100+ PRs by UX in a month
- Fabletics: $600K saved in productivity in the first few months (Publish)

## Competitors

| Category | Players | Our Edge |
|----------|---------|----------|
| AI Prototyping / App Builders | v0, Lovable, Bolt, Figma Make | Production-ready code with real design systems vs. demo-only throwaway |
| AI IDEs | Cursor, GitHub Copilot, Windsurf | Visual editing + AI accessible to non-devs vs. developer-only |
| AI Agents | Devin, Factory.ai | Collaborative workspace with human-in-the-loop vs. autonomous black-box |
| Traditional CMS | Contentful, Sanity, Storyblok, Adobe, Sitecore | Visual editing core to every tier, composable/headless, modern AI |

**Key differentiators:**
1. **Real component systems** — works with actual design system components, not generic ShadCN/Tailwind
2. **Visual + code flexibility** — both visual editing and code control, never forced to choose
3. **Production quality** — production-ready code with a11y, performance, enterprise requirements
4. **Ongoing iteration** — continuous visual editing after creation, not one-shot generation

**Positioning vs. Cursor:** "Builder isn't a replacement. It's the layer on top. Cursor makes one dev faster. Builder is about two things Cursor can't do: getting the rest of the team contributing directly to production code, and running massively in parallel on cloud environments instead of one agent at a time."

**Keywords to use:** Design system, visual editor for your code, design-to-code, enterprise-ready, drag-and-drop with code components.
**Keywords to avoid:** No-code, low-code, site builder, app builder.
